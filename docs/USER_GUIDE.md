## 1. ההיכרות הראשונה

לאחר `npm run dev` ופתיחה ב-http://localhost:5173, ייפתח **הטאב "שיחה"** במצב standard עם הודעת פתיחה.

מבנה המסך:
```
┌──────────────────────────────────────────────────────────┐
│ Header: לוגו · ONLINE · view-modes · autonomy · 🌞 · 👤  │
├────┬─────────────────────────┬──┬───────────────────────┤
│ S  │                         │↔ │                       │
│ i  │      Active Panel       │  │   Map (optional)      │
│ d  │      (משתנה לפי טאב)     │  │                       │
│ e  │                         │  │                       │
│ b  │                         │  │                       │
│ a  │                         │  │                       │
│ r  │                         │  │                       │
├────┴─────────────────────────┴──┴───────────────────────┤
│ StatusBar: LINK · SECURE · CPU · משימות · ⏰              │
└──────────────────────────────────────────────────────────┘
```

ב-Sidebar 6 פריטים: **שיחה · סוכנים · משימות · ציר זמן · אוטומציה · היסטוריה**. ה-badges הקטנים מציגים ספירה (סוכנים פעילים, משימות בביצוע, חוקים פעילים).

## 2. שיחה עם ה-AI

הודעת הפתיחה כוללת 6 **פעולות מהירות** (chips). לחיצה על כל chip שולחת prompt מוכן:
- *"תקציר 6 שעות אחרונות"*
- *"תכנן אש לצפון-מערב בשעה 15:00"*
- *"תקיפה משולבת אש+אוויר+מודיעין"*
- *"שגר כט"ב לצפון-מזרח"*
- *"מצב סוכנים"*
- *"איומים פעילים"*

ב-ChatInput בתחתית:
- שדה טקסט (Enter = שליחה, Shift+Enter = שורה חדשה)
- 3 כפתורי attachment בצד שמאל: 📍 / 🖼️ / 🎯
- כפתור "שלח"

בכותרת ה-ChatPanel: כפתור **"תקציר"** עם dropdown של 5 טווחי זמן (ראה סעיף 5).

הסימולטור AI מזהה keywords ומחזיר 7 סוגי תוכן:

| Prompt מילות מפתח | תוצר |
|------------------|------|
| `סטטוס + סוכנ` | טבלת מצב סוכנים |
| `איום / איתור / מתגלה / התרעה` | ActionCard + הצעת ישות חדשה למפה |
| `סכם / תקציר / דוח` | **SummaryBlock** מובנה |
| `אבחן / תקלה / שגיאה / בדוק` | ActionCard עם פרטי טעות |
| פעולה+דומיין (e.g. `תכנן אש`, `שגר כט"ב`) | **PlanProposal** |
| ברירת מחדל | טקסט + 4 chips של פעולות מהירות |

## 3. Flow תכנון: בקשה → אישור → משימה

הזרימה החשובה במערכת. דוגמה:

**שלב 1: בקשה**
שלח: *"תכנן הפעלת משימת אש לגזרת צפון-מערב בשעה 15:00"*

**שלב 2: ה-AI מנתח**
מציג הודעת מעבר: *"קלטתי את הבקשה. ניתחתי את ההקשר ובחרתי בסוכן אש... אזור הפעולה זוהה: גזרה צפון-מערב..."*

**שלב 3: PlanProposal**
מוצג כרטיס תוכנית גדול עם:
- Header עם הסוכן הנבחר + Badge "ממתינה לאישור" (pulse)
- נימוק "למה הסוכן הזה" — כולל success rate
- **multi-select של תתי-סוכנים** (chips של כל הסוכנים האחרים) — נסה לסמן
- 3 KPI: משך כולל / זמן הפעלה / עדיפות
- סקציית "אזור פעולה מוקצה" + מיני-מפה ימין עם פוליגון אדום מהבהב
- 5 צעדי ביצוע ממוספרים עם זמני הערכה
- flow ויזואלי: עכשיו → המתנה → הפעלה → סיום
- 3 כפתורים: ✓ אשר והפעל · ✏️ ערוך תוכנית · ✗ דחה

**שלב 4: עריכה (אופציונלי)**
לחיצה על "ערוך תוכנית":
- ניתן לשנות סוכן (Select)
- ניתן לסמן/לבטל תתי-סוכנים (chips)
- ניתן לשנות תאריך/שעה (datetime-local)
- ניתן לשנות עדיפות
- כל צעד הופך לעריכה inline (כותרת + דקות + ✕ מחיקה)
- כפתור "+ הוסף צעד"

**שלב 5: אישור**
לחיצה על "אשר והפעל":
- toast: *"המשימה אושרה ותופעל ב-15:00 (סוכן אש)"*
- הכרטיס משנה ל-"✓ אושרה ונשלחה"
- בפועל נוצרת `Task` חדשה: `status: 'planned'`, עם `scheduledFor`, ETA מחושב, `geo`, ו-`subAgentIds` אם נבחרו
- ה-Task מופיע מיד:
  - בטאב **משימות** תחת "מתוכננות"
  - בטאב **ציר זמן** כבלוק כתום ב-lane של סוכן האש
  - ב-AgentCard של סוכן האש — chip חדש "גזרה צפון-מערב"
  - ב-MockMap — פוליגון אדום עם תווית

## 4. משימות רב-סוכניות

מודל: **coordinator** (`agentId`) + **sub-agents** (`subAgentIds[]`). הקואורדינטור מתאם וה-subs מבצעים תתי-משימות.

**יצירה ב-PlanProposal**:
- שלח *"תכנן תקיפה משולבת של אש, אוויר ומודיעין בגזרה צפון-מזרחית בשעה 16:00"*
- ה-AI מזהה את 3 הדומיינים, בוחר את אש כקואורדינטור, ומסמן אוויר+מודיעין כ-subs מראש
- ב-PlanProposal תראה chip ירוק "מתאם · 2 תתי-סוכנים" ואת ה-chips מסומנים

**איך זה נראה אחרי אישור**:
- **TasksPanel**: שורת המשימה מציגה Badge "מתאם" ליד שם הסוכן + chips של תתי-הסוכנים
- **TimelinePanel**:
  - הבלוק הראשי ב-lane של ה-coordinator עם Badge `+2`
  - **בלוקים נוספים מקווקו דקים** (opacity 50%) ב-lanes של אוויר ומודיעין באותו טווח זמן, עם תווית *"↳ סוכן אש · תקיפה משולבת..."*
- **AgentCard / AgentMapPopover** של תת-סוכן: Badge "תת-סוכן ב-N משימות"
- **TaskDrawer** של תת-סוכן: רואה את המשימה עם בורדר כחול + Badge "תת-משימה" + שורת "מתואם ע"י: <coordinator>"

מומלץ לפתוח את ציר הזמן אחרי יצירה כזו כדי לראות את הקישור הוויזואלי בין ה-lanes.

## 5. תקצירי תקופה

יכולת ייעודית להציג תקציר מובנה כתשובת AI בצ'אט.

**3 דרכים להפיק**:
1. **Quick action בהודעת הפתיחה** — "תקציר 6 שעות אחרונות"
2. **כפתור "תקציר" בכותרת ChatPanel** — dropdown עם 5 טווחים (1h / 6h / 12h / 24h / 3d)
3. **טקסט חופשי** — *"סכם את 24 השעות האחרונות"* (ה-AI מזהה את המספר)

**מבנה התקציר** (`SummaryBlock`):
- **Header** ירוק זוהר + Badge "LIVE" + טווח זמן + כפתור "ייצא"
- **6 KPI cells** עם:
  - השלמות / יצירות / כשלונות
  - פעילות כעת
  - הפעלות חוקים
  - הצעות AI ממתינות
  - כל אחד עם trend (📈/📉/➖) ו-delta %
- **תובנות עיקריות** — bullets ירוקים, תומך bold
- **ציר אירועים** — אירועים sorted newest-first עם severity colors + agent chips
- **פעילות לפי סוכן** — קלוצים בגריד 2 עמודות עם mini bar (השלמות/בביצוע/נכשלו)
- **המלצות פעולה** — ממוספרות, ברקע warn + border-r כתום
- **Footer tabs פנימיים**: הכל / אירועים / סוכנים (סינון מהיר של התצוגה)

הנתונים מבוססים על ה-state **בפועל**, לא נתונים סטטיים — אם תיצור משימה חדשה ותפיק תקציר אחרי, היא תופיע בו.

## 6. ניהול סוכנים

לחץ על **"סוכנים"** ב-Sidebar. גריד **container-aware** (1/2/3 עמודות לפי רוחב הפאנל בפועל — לא viewport).

לכל כרטיס:
- אייקון דומיין במרובע
- שם + Badge סטטוס (פעיל/סרק/שגיאה/מנותק) עם pulse-dot
- 2 stats: מס' משימות פעילות + Select אוטונומיה
- שורת אזורי פעילות (אם יש משימות עם geo): chips של labels בצבע הדומיין
- 3 כפתורי פעולה:
  - **"פתח משימות"** → TaskDrawer
  - **📍** → פותח את המפה + מדגיש את הסוכן (השאר מעומעמים)
  - **⚙️** → AgentSettingsModal

## 7. שיחה ישירה עם סוכן

פותח דרך **TaskDrawer** של סוכן → לשונית **"שיחה"**.

מאפיינים:
- **Banner צבעוני** עליון: "שיחה ישירה עם <חתימת הסוכן>"
- **Empty state** מזמין: אווטר גדול + 4 quick action suggestions:
  - "מה המצב שלך?"
  - "אילו משימות פתוחות?"
  - "מי כפוף אליי?"
  - "יש לך המלצה לפעולה?"
- הסוכן עונה **בקול שלו** (signature שמוגדר ב-config.style)
- מודע למשימות שלו (כולל אלו שבהן הוא תת-סוכן)
- מודע ליחידות המקושרות וכלים זמינים
- כפתור איפוס שיחה בפינה
- ה-thread נשמר ב-state (לא נמחק עד reload או איפוס מפורש)

**דוגמאות לבדיקה**:
- פתח שיחה עם סוכן אש → "אילו משימות פתוחות?" → תקבל טבלת המשימות שלו
- שאל "מה המצב?" → תקבל סטטוס דומיין-ספציפי (פלוגות, תחמושת)
- שאל "מי כפוף אליי?" → טבלת היחידות המקושרות

## 8. הגדרות סוכן

נגיש דרך ⚙️ ב-AgentCard או ב-AgentMapPopover.

מודאל xl עם sidebar שמאלי + 6 לשוניות:

### כללי
- 3 KPIs גדולים (דומיין/סטטוס/הצלחה)
- 6 stat boxes (autonomy, משימות, יחידות, כלים, ביצוע אוטומטי, תיחום)

### סגנון תקשורת
- **חתימה** - השם שמופיע בצ'אט
- **טון**: רשמי / טקטי-מבצעי / תמציתי / מפורט
- **רמת פירוט**: מינימלי / סטנדרטי / מורחב
- **שפה**: עברית / אנגלית / מעורב
- **שימוש באימוג'י** (toggle)
- **ציטוט מקורות** (toggle)
- **תצוגה מקדימה חיה** של דוגמת הודעה — מתעדכנת בזמן אמת

### הרשאות
- 6 capability toggles: יצירת משימות / הקצאת יחידות / הסלמת התרעות / שינוי ישויות / גישה למסווג / ביצוע ללא אישור
- סף עדיפות לאישור אוטומטי (never/low/medium/high)
- מקסימום משימות מקבילות
- אזהרה אדומה אוטומטית כשמופעל "ביצוע ללא אישור"

### יחידות
- רשימת יחידות מקושרות עם type ו-callsign
- "קשר יחידה חדשה" - שם / סוג / קריאה

### תיחום
- שמאל: שם + סוג + שטח משוער (shoelace formula) + bounding box
- **כפתור "ערוך תיחום במפה"** — פותח MapPickerModal **מעל** ההגדרות, עם הפוליגון הקיים נטען מראש
- ניתן: הוספת נקודות, ביטול נקודה אחרונה, איפוס, "סיים תיחום"
- בסיום: ה-מיני-מפה במודאל ההגדרות מתעדכנת, dirty סומן
- empty state עם 2 כפתורים: "סמן תיחום במפה" או "מלבן ברירת מחדל"

### כלים
- רשימת AgentTool עם toggle הפעל/השבת
- "צור כלי חדש" - שם / תיאור / סוג (Webhook/שאילתה/פעולה/ניתוח) / endpoint URL (אם webhook) / toggle הפעל מיידית

**שמירה**:
- כל שינוי מסומן כ-dirty (חיווי כתום בפוטר)
- "שמור שינויים" disabled עד שיש dirty
- סגירה עם dirty → confirm dialog

## 9. תצוגת משימות

לחץ על **"משימות"** ב-Sidebar.

מבנה:
- **5 KPI buttons** למעלה - בביצוע / מתוכננות / ממתינות / הושלמו / נכשלו. ספירה גדולה ולחיצה מסננת.
- **שורת סינון**: chips + dropdown סוכן + חיפוש טקסט
- **גוף**: כשבחור "הכל" המשימות מקובצות לפי סטטוס עם separators

לכל TaskRow:
- **כל הכרטיס clickable** → פותח TaskDetailsModal (סעיף 10)
- בר צבעוני אנכי בצד (לפי סטטוס)
- אייקון דומיין + כותרת + 2 badges
- meta line: Badge "מתאם" אם רב-סוכנית + שם הסוכן + **chip של geo קליקבילי** (`stopPropagation` — לא פותח את ה-modal, אלא פותח את המפה + highlight) + chips של תתי-סוכנים + זמנים
- progress bar
- כפתור "בטל" (רק לפעילות, `stopPropagation`)

## 10. פירוט ועריכת משימה

נפתח בכל לחיצה על משימה (מ-TasksPanel או מבלוק בציר הזמן).

**Sidebar שמאלי (240px)** — תמצית במבט אחד:
- task id
- 2 badges (סטטוס + עדיפות) עם pulse לבביצוע
- **כרטיס סוכן אחראי** בצבע הדומיין (כותרת מתחלפת ל"מתאם" אם רב-סוכנית)
- preview של תתי-סוכנים
- progress bar
- 5 timestamps (נוצרה / מתוזמן / החלה / ETA / הושלמה) — מודגשים בצבע סמנטי

**טור ימני (form)** — toggle צפייה/עריכה:
- כותרת + תיאור
- 3 selects: סטטוס / עדיפות / סוכן אחראי
- 2 שדות datetime: scheduledFor + ETA
- **multi-select של תתי-סוכנים** (chips בצבע דומיין)
- **אזור גיאוגרפי** עם מיני-מפה ימין + כפתורי "ערוך במפה" / "הסר"
- שורת hint למטה

**פעולות**:
- **שמור שינויים** (disabled עד dirty)
- **בטל משימה** (confirm → CANCEL_TASK → סוגר; אדום)
- **סמן כהושלמה** (רק ל-in_progress; קובע completedAt+progress=100)
- **סגור** (confirm על שינויים לא שמורים)
- משימות **completed/failed** הופכות לקריאה בלבד עם Badge

**עריכת geo במפה**: לחיצה על "ערוך במפה" פותחת MapPickerModal עם הפוליגון/נקודה הקיימים נטענים מראש. אותו flow כמו עריכת תיחום סוכן.

## 11. ציר זמן מבצעי

לחץ על **"ציר זמן"**. תצוגת Gantt-style עם:

```
            [4h] [12h] [24h] [3d]    ← זום
┌────────┬──────────────────────────────┐
│ סוכן  │  14:00   15:00   16:00   ●עכשיו │  ← header עם ticks
├────────┼──────────────────────────────┤
│ 🔥 אש  │   ████████░░░░░░             │  ← lane (one per agent)
│ 👁 מוד │   ░░░░████████░░  ↳אש·משולב  │  ← sub-agent indicator (dashed)
│ ✈️ אוויר│           ████████░░         │
│ ...   │                              │
└────────┴──────────────────────────────┘
            cyan vertical line = NOW
```

**אינטראקציות**:
- **לחיצה על בלוק** → פותח TaskDetailsModal (גם בלן ראשי וגם תת-משימה מקווקו)
- כפתורי "הקדם / עכשיו / אחר" - הזזת חלון הזמן (25% מהרוחב)
- בחירת זום (4h / 12h / 24h / 3d) - מתעדכן מיידית
- hover על בלוק - מציג פירוט בלגנדה התחתונה
- בלוק "in_progress" מציג progress bar קטן בתחתית
- בלוק רב-סוכני מציג Badge `+N`
- ספירת lane: "5 בחלון · +2 תת" (משתתפויות כתת-סוכן)
- צבעים: כחול=בביצוע, כתום=מתוכננת, אפור=ממתינה, ירוק=הושלמה, אדום=נכשלה, מקווקו=תת-משימה

## 12. חוקי אוטומציה

לחץ על **"אוטומציה"**.

**פאנל "לוגיקת אוטומציה כוללת"** (פתוח ב-default) למעלה:
- 3 KPIs: פעולות אוטומטיות / ידרשו אישור / התנגשויות
- עומס לפי סוכן (chips, אדום אם ≥3 חוקים)
- "מה יקרה — חוק אחר חוק" — רשימה ממוספרת עם משפט דינמי
- התנגשויות אוטומטיות (duplicate trigger / agent overload)

**רשימת RuleCards** למטה. לכל אחת:
- Toggle הפעל/השבת
- שם + Badge "שפה חופשית" אם נוצרה מ-NL + ציטוט המקור באיטליקה
- WHEN ← THEN בעיצוב טקטי
- ספירת הפעלות + כפתורי ערוך/מחק

**יצירת חוק חדש** (RuleEditor):

**Mode toggle**: "הגדרה מובנית" ⟷ "שפה חופשית"

### מצב שפה חופשית
- Textarea + 3 דוגמאות מהירות
- כתוב: *"איתור איום בצפון → הקצה סוכן אש בעדיפות גבוהה"* או *"כל 4 שעות צור משימת ניטור למודיעין"*
- לחץ "פענח ועדכן" → השדות המובנים מתמלאים
- Feedback מתחת: ✓ ירוקים (מה הובן) + ⚠ כתומים (מה חסר/לא ברור)

### מצב מובנה
- **WHEN**: 3 סוגי טריגר:
  - **איתור בשטח**: סוג ישות + תיאור אזור + כפתור "בחר תיחום במפה"
  - **חציית סף**: מטריקה + אופרטור + ערך
  - **תזמון**: recurrence (חד-פעמי / כל שעה / יומי / שבועי) + שדות תלויי-בחירה + תצוגת "הפעלה הבאה משוערת"
- **THEN**: סוג פעולה / סוכן יעד / עדיפות / הודעה

**בלוק "מה ה-AI יעשה?"** למטה — בורדר ירוק זוהר, משפט מסכם מ-`describeRuleLogic()` שלוקח בחשבון autonomy גלובלית + סוכנית, ומסביר אם הפעולה תרוץ אוטומטית או תדרוש אישור.

## 13. המפה ו-popovers

המפה זמינה ב-2 דרכים:
- **expanded view mode**: היא תמיד מוצגת לצד הפאנל הפעיל
- **כפתור 🗺️ ב-Header** (במצב standard/compact)

**Splitter**: גרור את הקו האנכי בין הפאנל למפה (4px עם grip indicator) כדי לשנות את חלוקת הרוחב. double-click מאפס ל-45%.

**הצגה**:
- רשת טקטית עם sectors (צפונית / צפון-מזרח / דרומית)
- terrain (נחל כחול, רכס כתום)
- מצפן + קואורדינטות
- scanline מהבהבת (אווירה)
- ישויות (איומים אדומים, יחידות כחולות, נכסים ירוקים, POI כתומים) - hoverable
- **אזורי פעילות סוכנים** (polygons/points בצבעי דומיין) - **קליקביליים**

**MapAgentLegend** מתחת למפה: chip לכל סוכן עם אזורי פעילות. לחיצה מחילה filter (מעמעמת שאר הסוכנים).

**AgentMapPopover** (לחיצה על אזור):
- Header עם רקע בצבע הדומיין + שם הסוכן + סטטוס + autonomy + last activity
- שורת אזור + Badges "מתאם N משימות"/"תת-סוכן ב-N משימות"
- 4 KPIs - ספירה ספציפית לאזור
- רשימת משימות באזור (עד 6)
- 3 כפתורי פעולה: **"פתח משימות"** / ⚙️ / 🎯

**MapPickerModal** נפתח כש:
- בצ'אט לחצת על 📍 (צרף תיחום) → פוליגון
- לחצת על "ערוך מיקום במפה" בכרטיס EntitySuggestion → נקודה
- ב-RuleEditor לחצת על "בחר תיחום" → פוליגון (placeholder)
- ב-AgentSettings בלשונית תיחום → פוליגון, נטען עם הקיים
- ב-TaskDetailsModal לחצת על "ערוך במפה" → פוליגון/נקודה, נטען עם הקיים

## 14. Theme, view modes, ו-splitter

### Theme toggle
לחץ **☀️/🌙** ב-Header. מעבר בין:
- **Dark** (ברירת מחדל) - terminal טקטי כהה עם ירוק זוהר
- **Light** - command desk בהיר עם ירוק כהה לקונטרסט

כל הקומפוננטות + המפה + ה-shadows + ה-grid background מתעדכנים מיידית.

### View modes
3 כפתורים ב-Header:
- **דחוס** - האפליקציה ב-`max-w-640px` במרכז. נוח לתצוגה צידית
- **רגיל** (ברירת מחדל) - האפליקציה ממלאת את הרוחב
- **מורחב** - כמו רגיל + המפה תמיד מוצגת לצד הפאנל

הכפתור 🗺️ מופיע רק במצב דחוס/רגיל (במורחב המפה מוצגת תמיד).

### Splitter
כשהמפה פתוחה — גרור את הקו האנכי לכוון את חלוקת הרוחב בין הפאנל למפה. double-click מאפס. הגריר רספונסיבי — ה-AgentsPanel מתעדכן בזמן אמת (1/2/3 עמודות לפי הרוחב הפנימי שלו).

## 15. רמות אוטונומיה

ב-Header יש Selector "אוטונומיה" עם 3 ערכים:

| רמה | משמעות | חיווי |
|------|---------|--------|
| **תצפית בלבד** | ה-AI רק מציג מידע, ללא המלצות לפעולה | Badge אפור |
| **המלצה לפעולה** | ה-AI מציע פעולות; דרוש אישור אנושי | Badge כחול |
| **פעולה אוטומטית** | ה-AI יכול לבצע פעולות ללא אישור | Badge כתום |

**הרמה הגלובלית** משפיעה על:
- Badge בכל הודעת AI (שקיפות)
- ההסבר ב-RuleEditor: *"החוק יופעל ברמת אוטונומיות גלובלית: X"*
- ה-PlanProposal מציג את הרמה הנוכחית
- "מה ה-AI יעשה?" — קובע אם הפעולה תוצג כהמלצה או תרוץ אוטומטית

**הרמה הסוכנית** ב-AgentCard (Select) — מאפשרת לעקוף את הגלובלית לסוכן ספציפי.

ב-AgentSettings יש שדה נוסף "סף עדיפות לאישור אוטומטי" — אפילו בסוכן autonomous, פעולות מעל לסף יחזרו לאישור אנושי.

## 16. Real LLM Mode

כל ה-walkthroughs לעיל עובדים גם ב-mock וגם ב-real LLM. ההבדל היחיד הוא מאיפה הבלוקים באים.

### הפעלה
```powershell
cp server/.env.example server/.env
# ערוך .env: LLM_BASE_URL, LLM_API_KEY, LLM_MODEL

cp .env.example .env
# שנה: VITE_USE_REAL_LLM=true

npm run dev
```

### הבדלים בחוויה
- **תשובות יצירתיות יותר** — ה-LLM יכול להבין בקשות חופשיות שה-mock לא מזהה (e.g. *"מה דעתך על תקיפה דו-שלבית?"*)
- **Streaming הדרגתי** — כל block מופיע כשהוא מוכן (לא הכל בבת אחת)
- **תוויות בעברית מורכבת יותר** — ה-LLM מנסח חופשי, לא טמפלטים
- **המשימות עצמן והגדרות נשארות זהות** — ה-LLM רק מייצר את התשובות

### בדיקות מהירות
- שלח prompt מעורפל: *"מה כדאי לעשות עכשיו?"* → ה-mock יחזיר תשובה גנרית, ה-LLM ינתח את ה-state ויציע פעולה רלוונטית
- שלח *"איזה סוכן עומס יותר מדי?"* → ה-LLM יקרא את המספרים ויחשב, ה-mock לא יבין
- כבה את ה-backend באמצע השיחה → תקבל toast אזהרה ו-fallback אוטומטי ל-mock

### מתי לא להשתמש
- הדגמות שבהן רוצים תגובות צפויות → mock עדיף
- ללא חיבור לאינטרנט (אלא אם משתמשים ב-Ollama מקומי) → mock עדיף
- לבדיקת UX של renderers → mock נותן שליטה מלאה

## דגשי תפעול וטיפים

### הצ'אט הוא נקודת המוצא הטבעית
רוב התרחישים מתחילים בצ'אט. שלח prompt ונחש איזה renderer יופיע. ה-quick actions בהודעת הפתיחה הם תפריט טוב להיכרות.

### Flow אישור = הדרך הנכונה ליצור משימה
אל תיצור משימה ידנית. תמיד עבור דרך *"תכנן..."* בצ'אט → אישור. זה מבטיח שהמשימה תהיה עם geo, schedule, ו-priority נכונים.

### המפה היא תמונת אמת
האזורים המוצגים = משימות אקטיביות. כשמשימה מסתיימת או מתבטלת, האזור נעלם.

### Highlight agent זה filter
לחיצה על chip בלגנדה או על 📍 בכרטיס סוכן — מצב ה-highlight נשמר עד שתבטל ידנית ("נקה סינון") או תלחץ על סוכן אחר.

### Cross-modal flows
ערכים שעוברים בין מודאלים (תיחום סוכן, geo של משימה) משתמשים ב-mailbox transient ב-state. המשמעות: גם אם תסגור את ה-picker בלי "סיים", השינוי לא יחול — צריך ללחוץ "סיים" → "שמור שינויים".

### דברים שלא עובדים (mock limitations)
- "בחר תיחום במפה" ב-RuleEditor — toast הסבר אבל אין בחירה אמיתית
- שיחות היסטוריות (HistoryPanel) — לחיצה מציגה toast אבל לא טוענת תוכן
- ה-Action card buttons ("אשר", "הסלם") — מציגים toast אבל לא משנים state
- כפתור הפעמון, ה-classification tag, ספירות CPU ב-StatusBar — קישוטים
- ייצוא PDF של תקציר — toast הדגמה
- Real LLM mode: אין persistence של היסטוריית שיחה — כל reload מאתחל

מה ש**כן** עובד מקצה-לקצה: יצירת משימה דרך PlanProposal (כולל multi-agent), עריכת משימה ב-TaskDetailsModal, ביטול משימה, חוקי אוטומציה (toggle + CRUD + NL parsing), שינוי הגדרות סוכן (כולל תיחום במפה אמיתית), EntitySuggestion → אישור + עריכת מיקום, attachments לצ'אט, ניווט בין כל הטאבים, החלפת theme/view-mode/autonomy, splitter, שיחה ישירה עם סוכן, תקצירים דינמיים.
