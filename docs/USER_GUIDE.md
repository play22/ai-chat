## 1. ההיכרות הראשונה

לאחר `npm run dev` ופתיחה ב-http://localhost:5173, ייפתח **הטאב "שיחה"** במצב standard עם הודעת פתיחה.

מבנה המסך:
```
┌──────────────────────────────────────────────────────────┐
│ Header: לוגו · ONLINE · view-modes · autonomy · 🌞 · 👤  │
├────┬─────────────────────────────────────────────────────┤
│ S  │                                                      │
│ i  │              Active Panel                            │
│ d  │              (משתנה לפי טאב)                          │
│ e  │                                                      │
│ b  │                                                      │
│ a  │                                                      │
│ r  │                                                      │
├────┴─────────────────────────────────────────────────────┤
│ StatusBar: LINK · SECURE · CPU · משימות · ⏰              │
└──────────────────────────────────────────────────────────┘
```

ב-Sidebar 6 פריטים: **שיחה · סוכנים · משימות · ציר זמן · אוטומציה · היסטוריה**. ה-badges הקטנים מציגים ספירה (סוכנים פעילים, משימות בביצוע, חוקים פעילים).

## 2. שיחה עם ה-AI

הודעת הפתיחה כוללת 5 **פעולות מהירות** (chips). לחיצה על כל chip שולחת prompt מוכן:
- *"תכנן אש לצפון-מערב בשעה 15:00"*
- *"הפק דו"ח מודיעין"*
- *"שגר כט"ב לצפון-מזרח"*
- *"תיגבור דלק לציר המרכזי"*
- *"מצב סוכנים"*

ב-ChatInput בתחתית:
- שדה טקסט (Enter = שליחה, Shift+Enter = שורה חדשה)
- 3 כפתורי attachment בצד שמאל: 📍 / 🖼️ / 🎯
- כפתור "שלח" כתום-ירוק

הסימולטור AI מזהה keywords ומחזיר תשובה בסגנונות שונים:

| Prompt מילות מפתח | תוצר |
|------------------|------|
| `סטטוס + סוכנ` | טבלת מצב סוכנים |
| `איום / איתור / מתגלה / התרעה` | ActionCard + הצעת ישות חדשה למפה |
| `סכם / תקציר / דוח` | טקסט מפורט + פעולות מהירות |
| `אבחן / תקלה / שגיאה / בדוק` | ActionCard עם פרטי טעות |
| פעולה+דומיין (e.g. `תכנן אש`, `שגר כט"ב`) | **PlanProposal** (ראה סעיף 3) |
| ברירת מחדל | טקסט + 4 chips של פעולות מהירות |

## 3. Flow תכנון: בקשה → אישור → משימה

הזרימה החשובה במערכת. דוגמה:

**שלב 1: בקשה**
שלח: *"תכנן הפעלת משימת אש לגזרת צפון-מערב בשעה 15:00"*

**שלב 2: ה-AI מנתח**
מציג הודעת מעבר: *"קלטתי את הבקשה. ניתחתי את ההקשר ובחרתי בסוכן אש... אזור הפעולה זוהה: גזרה צפון-מערב..."*

**שלב 3: PlanProposal**
מוצג כרטיס תוכנית גדול עם:
- **Header**: סוכן נבחר + Badge "ממתינה לאישור" (pulse)
- **נימוק "למה הסוכן הזה"** — כולל success rate
- **3 KPI**: משך כולל / זמן הפעלה / עדיפות
- **סקציית "אזור פעולה מוקצה"** + מיני-מפה ימין עם פוליגון אדום מהבהב
- **5 צעדי ביצוע** ממוספרים עם זמני הערכה
- **flow ויזואלי**: עכשיו → המתנה → הפעלה → סיום
- **3 כפתורים**: ✓ אשר והפעל · ✏️ ערוך תוכנית · ✗ דחה

**שלב 4: עריכה (אופציונלי)**
לחיצה על "ערוך תוכנית":
- ניתן לשנות סוכן (Select)
- ניתן לשנות תאריך/שעה (datetime-local)
- ניתן לשנות עדיפות
- כל צעד הופך לעריכה inline (כותרת + דקות + ✕ מחיקה)
- כפתור "+ הוסף צעד"

**שלב 5: אישור**
לחיצה על "אשר והפעל":
- toast: *"המשימה אושרה ותופעל ב-15:00 (סוכן אש)"*
- הכרטיס משנה ל-"✓ אושרה ונשלחה"
- בפועל נוצרת `Task` חדשה: `status: 'planned'`, עם `scheduledFor`, ETA מחושב, ו-`geo`
- ה-Task מופיע מיד:
  - בטאב **משימות** תחת "מתוכננות"
  - בטאב **ציר זמן** כבלוק כתום ב-lane של סוכן האש, ממוקם בשעה 15:00
  - ב-AgentCard של סוכן האש — chip חדש "גזרה צפון-מערב" באזורי הפעילות
  - ב-MockMap — פוליגון אדום עם תווית

**דוגמאות נוספות לבדיקה**:
- *"תכנן שיגור כט"ב לסיור בגזרה הצפון-מזרחית בשעה 14:30"* → סוכן אוויר
- *"תכנן הפקת דו"ח מודיעין מעודכן בעוד 30 דקות"* → סוכן מודיעין
- *"תכנן תיגבור דלק לטור בציר המרכזי בעוד שעה"* → סוכן לוגיסטיקה
- *"בצע תקיפת רשת קריטית"* → סוכן סייבר, עדיפות critical

המערכת מזהה אוטומטית: דומיין (לפי keywords), זמן (HH:MM / `בשעה X` / `בעוד N דקות`), אזור (`צפון-מערב`, `נמל`, `ציר`...), ועדיפות (`קריטי` / `דחוף` / `גבוה`).

## 4. ניהול סוכנים

לחץ על **"סוכנים"** ב-Sidebar. רואים גריד של 6 כרטיסים.

לכל כרטיס:
- **אייקון דומיין** במרובע
- **שם + Badge סטטוס** (פעיל/סרק/שגיאה/מנותק) עם pulse-dot
- **2 stats**: מס' משימות פעילות (גדול, ירוק) + Select אוטונומיה
- **שורת אזורי פעילות** (אם יש משימות עם geo): chips של labels בצבע הדומיין
- **3 כפתורי פעולה**:
  - **"פתח משימות"** → drawer צד עם רשימת כל המשימות של הסוכן
  - **📍** → פותח את המפה + מדגיש את הסוכן הזה (השאר מעומעמים)
  - **⚙️** → פותח את ה-AgentSettingsModal (ראה סעיף 9)

**TaskDrawer**:
פותח drawer מצד שמאל עם:
- כותרת הסוכן
- summary: סה"כ משימות + כמה בביצוע
- רשימת משימות, לכל אחת: כותרת, badges (priority, status), timestamps, progress bar, כפתור "בטל משימה" (לא לסיומות)

## 5. תצוגת משימות

לחץ על **"משימות"** ב-Sidebar.

מבנה:
- **5 KPI buttons** למעלה - בביצוע / מתוכננות / ממתינות / הושלמו / נכשלו. כל אחד מציג ספירה גדולה ולחיצה מסננת.
- **שורת סינון**: chips + dropdown סוכן + חיפוש טקסט
- **גוף**: כשבחור "הכל" המשימות מקובצות לפי סטטוס עם separators

לכל TaskRow:
- בר צבעוני אנכי בצד (לפי סטטוס)
- אייקון דומיין + כותרת + 2 badges
- **chip קליקבילי של geo** עם 📍 ושם האזור → לוחצים פותח המפה + highlight
- meta line: סוכן, מתוזמן/החל/הושלם/ETA
- progress bar (אם רלוונטי)
- כפתור "בטל" (רק לפעילות)

**טיפ**: סנן ל"מתוכננות" כדי לראות את כל המשימות העתידיות, כולל אלו שזה עתה נוצרו מהצ'אט.

## 6. ציר זמן מבצעי

לחץ על **"ציר זמן"**. תצוגת Gantt-style עם:

```
            [4h] [12h] [24h] [3d]    ← זום
┌────────┬──────────────────────────────┐
│ סוכן  │  14:00   15:00   16:00   ●עכשיו │  ← header עם ticks
├────────┼──────────────────────────────┤
│ 🔥 אש  │   ████████░░░░░░             │  ← lane lev (one per agent)
│ 👁 מוד │   ░░░░████████░░             │
│ ✈️ אוויר│           ████████░░         │
│ ...   │                              │
└────────┴──────────────────────────────┘
            cyan vertical line = NOW
```

**אינטראקציות**:
- כפתורי "הקדם / עכשיו / אחר" - הזזת חלון הזמן (25% מהרוחב כל לחיצה)
- בחירת זום (4h / 12h / 24h / 3d) - מתעדכן מיידית
- hover על בלוק - מציג פירוט בלגנדה התחתונה
- בלוק "in_progress" מציג progress bar קטן בתחתית
- שמות הצבעים: כחול=בביצוע, כתום=מתוכננת, אפור=ממתינה, ירוק=הושלמה, אדום=נכשלה

## 7. חוקי אוטומציה

לחץ על **"אוטומציה"**. רואים רשימת חוקים קיימים + כפתור "חוק חדש".

לכל RuleCard:
- Toggle הפעל/השבת (שמאל)
- שם החוק
- Badge "פעיל" עם pulse או "מושבת"
- **WHEN ← THEN**: תיאור הטריגר → תיאור הפעולה, עם אייקון הסוכן ועדיפות
- footer: ספירת הפעלות, זמן אחרון, כפתורי ערוך/מחק

**יצירת חוק חדש** (RuleEditor):
מודאל גדול עם 2 טורים.

**WHEN** (טריגר) - בחירת סוג מתוך 3:
- **איתור בשטח**: סוג ישות + תיאור אזור + כפתור "בחר תיחום במפה"
- **חציית סף**: מטריקה + אופרטור + ערך מספרי
- **תזמון**: בחירת recurrence (חד-פעמי / כל שעה / יומי / שבועי) + שדות תלויי-בחירה (datetime-local / מרווח שעות / שעה / יום+שעה) + **תצוגת "הפעלה הבאה משוערת"** דינמית

**THEN** (פעולה):
- סוג פעולה (צור משימה / שלח התרעה / הקצה יחידה)
- סוכן יעד
- עדיפות
- הודעה/תיאור

לחיצה על "צור חוק" → toast הצלחה, חוק מופיע ברשימה.

## 8. המפה

המפה זמינה ב-2 דרכים:
- **expanded view mode**: היא תמיד מוצגת לצד הפאנל הפעיל
- **כפתור 🗺️ ב-Header** (במצב standard/compact)

לאחר שמופעלת, מציגה:
- **רשת טקטית** עם sectors (צפונית / צפון-מזרח / דרומית)
- **terrain** (נחל כחול, רכס כתום)
- **מצפן** + קואורדינטות
- **scanline** מהבהבת (אווירה)
- **ישויות** (איומים אדומים, יחידות כחולות, נכסים ירוקים, POI כתומים) - hoverable
- **אזורי פעילות סוכנים** (polygons/points בצבעי דומיין) - **קליקביליים**

**MapAgentLegend** מתחת למפה: chip לכל סוכן עם אזורי פעילות. לחיצה מחילה filter (מעמעמת שאר הסוכנים).

**AgentMapPopover** (לחיצה על אזור):
- Header עם רקע בצבע הדומיין + שם הסוכן + סטטוס + autonomy + last activity
- שורת אזור: "אזור: גזרה צפון-מערב"
- 4 KPIs - ספירה ספציפית לאזור
- רשימת משימות באזור (עד 6)
- 3 כפתורי פעולה:
  - **"פתח משימות"** → TaskDrawer של כל המשימות של הסוכן
  - **⚙️** → AgentSettingsModal
  - **🎯** → נווט לטאב סוכנים

**MapPickerModal** נפתח כש:
- בצ'אט לחצת על 📍 (צרף תיחום) → פוליגון
- לחצת על "ערוך מיקום במפה" בכרטיס EntitySuggestion → נקודה
- ב-RuleEditor לחצת על "בחר תיחום" → פוליגון

## 9. הגדרות סוכן

נגיש דרך ⚙️ ב-AgentCard או ב-AgentMapPopover.

מודאל xl עם sidebar שמאלי + 6 לשוניות:

### כללי
- 3 KPIs גדולים (דומיין/סטטוס/הצלחה)
- 6 stat boxes (autonomy, משימות, יחידות, כלים, ביצוע אוטומטי, תיחום)

### סגנון תקשורת
- **חתימה** - השם שמופיע בצ'אט
- **טון**: רשמי / טקטי-מבצעי / תמציתי / מפורט
- **רמת פירוט**: מינימלי / סטנדרטי / מורחב
- **שפה**: עברית / אנגלית / מעורב
- **שימוש באימוג'י** (toggle)
- **ציטוט מקורות** (toggle)
- **תצוגה מקדימה חיה** של דוגמת הודעה — מתעדכנת בזמן אמת

### הרשאות
- 6 capability toggles בגריד: יצירת משימות / הקצאת יחידות / הסלמת התרעות / שינוי ישויות / גישה למסווג / ביצוע ללא אישור
- סף עדיפות לאישור אוטומטי (never/low/medium/high)
- מקסימום משימות מקבילות
- **אזהרה אדומה** אוטומטית כשמופעל "ביצוע ללא אישור"

### יחידות
- רשימת יחידות מקושרות עם type ו-callsign
- "קשר יחידה חדשה" - שם / סוג / קריאה

### תיחום
- שמאל: שם + סוג + כפתורי "ערוך תיחום במפה" / "הסר"
- ימין: מיני-מפה עם previewGeo של התיחום הנוכחי
- empty state עם "הוסף תיחום"

### כלים
- רשימת AgentTool עם toggle הפעל/השבת
- "צור כלי חדש" - שם / תיאור / סוג (Webhook/שאילתה/פעולה/ניתוח) / endpoint URL (אם webhook) / toggle הפעל מיידית

**שמירה**:
- כל שינוי מסומן כ-dirty (חיווי כתום בפוטר)
- "שמור שינויים" disabled עד שיש dirty
- סגירה עם dirty → confirm dialog
- שינויי הconfig **לא משפיעים בעיצוב או התנהגות** של ההדגמה (mock), אבל נשמרים ב-state ומתועדים ב-AgentCard (e.g. אזורי תיחום).

## 10. Theme & view modes

### Theme toggle
לחץ **☀️/🌙** ב-Header (ליד הפעמון). מעבר בין:
- **Dark** (ברירת מחדל) - terminal טקטי כהה עם ירוק זוהר
- **Light** - command desk בהיר עם ירוק כהה לקונטרסט

כל הקומפוננטות + המפה + ה-shadows + ה-grid background מתעדכנים מיידית.

### View modes
3 כפתורים ב-Header:
- **דחוס** - האפליקציה ב-`max-w-640px` במרכז. נוח לתצוגה צידית
- **רגיל** (ברירת מחדל) - האפליקציה ממלאת את הרוחב
- **מורחב** - כמו רגיל + המפה תמיד מוצגת לצד הפאנל בחלוקה 55/45

הכפתור 🗺️ הופיע רק במצב דחוס/רגיל (כי במורחב המפה מוצגת תמיד).

## 11. רמות אוטונומיה

ב-Header יש Selector "אוטונומיה" עם 3 ערכים:

| רמה | משמעות | חיווי |
|------|---------|--------|
| **תצפית בלבד** | ה-AI רק מציג מידע, ללא המלצות לפעולה | Badge אפור |
| **המלצה לפעולה** | ה-AI מציע פעולות; דרוש אישור אנושי לכל ביצוע | Badge כחול |
| **פעולה אוטומטית** | ה-AI יכול לבצע פעולות ללא אישור | Badge כתום |

**הרמה הגלובלית** משפיעה על:
- Badge בכל הודעת AI (שקיפות - לראות באיזו רמה ניתנה)
- ההסבר ב-RuleEditor: *"החוק יופעל ברמת אוטונומיות גלובלית: X"*
- ה-PlanProposal מציג את הרמה הנוכחית

**הרמה הסוכנית** ב-AgentCard (Select) — מאפשרת לעקוף את הגלובלית לסוכן ספציפי. למשל גלובלית="המלצה" אבל סוכן אוויר="אוטונומי".

ב-AgentSettings יש שדה נוסף "סף עדיפות לאישור אוטומטי" - אפילו בסוכן autonomous, פעולות מעל לסף יחזרו לאישור אנושי.

## דגשי תפעול וטיפים

### הצ'אט הוא הנקודת מוצא הטבעית
רוב התרחישים מתחילים בצ'אט. שלח prompt ונחש איזה renderer יופיע. ה-quick actions בהודעת הפתיחה הם תפריט טוב להיכרות.

### Flow אישור = הדרך הנכונה ליצור משימה
אל תיצור משימה ידנית. תמיד עבור דרך *"תכנן..."* בצ'אט → אישור. זה מבטיח שהמשימה תהיה עם geo, schedule, ו-priority נכונים.

### המפה היא תמונת אמת
האזורים המוצגים = משימות אקטיביות. כשמשימה מסתיימת או מתבטלת, האזור נעלם. זה מאפשר תמונת מצב נכונה בכל רגע.

### Highlight agent זה filter
בחיצה על chip בלגנדה או על 📍 בכרטיס סוכן — מצב ה-highlight נשמר עד שתבטל ידנית ("נקה סינון") או תלחץ על סוכן אחר.

### דברים שלא עובדים (mock limitations)
- "בחר תיחום במפה" ב-RuleEditor — toast הסבר אבל אין בחירה אמיתית (זה משאיר את הbar הוא קצר; השדה areaLabel הוא טקסטואלי).
- שיחות היסטוריות (HistoryPanel) — לחיצה מציגה toast אבל לא טוענת תוכן אמיתי.
- ה-Action card buttons ("אשר", "הסלם", וכו') — מציגים toast אבל לא משנים state. הם דמו ל-UI.
- כפתור הפעמון, ה-classification tag, ספירות ה-CPU ב-StatusBar — קישוטים.

מה ש**כן** עובד מקצה-לקצה: יצירת משימה דרך PlanProposal, חוקי אוטומציה (toggle + CRUD), שינוי הגדרות סוכן, תיחום סוכן במפה, EntitySuggestion → אישור + עריכת מיקום, attachments לצ'אט, ניווט בין כל הטאבים, החלפת theme/view-mode/autonomy.
